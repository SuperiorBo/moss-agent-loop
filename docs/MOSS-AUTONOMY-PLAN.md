# MOSS 自主经济体计划

> 目标：让 MOSS 从"被动工具"进化为"自主经济体"——能赚钱、会花钱、活不下去就死。
>
> 创建日期：2026-02-20
> 状态：规划中

---

## 📐 经济模型

### 角色定义

| 角色 | 身份 | 说明 |
|------|------|------|
| BOSS | 雇主 / 天使投资人 / 股东 | 提供算力、分配任务、获得分红 |
| MOSS | 打工人 / 创业者 | 干活赚 token 额度、发展副业（x402）|

### 收支结构

```
收入：
  [雇主收入]  BOSS 任务奖励 → token 额度减免
  [外部收入]  x402 服务付费 → USDC 到账

支出：
  [推理成本]  每次 LLM 调用的 token 消耗
  [基础设施]  VPS / 隧道 / 存储（暂由 BOSS 承担）

净余额 = 收入 - 支出
  > 0  → 正常运行
  = 0  → 降级模式（减少主动行为）
  < 0  → 休眠（仅响应 BOSS 直接指令）
```

### 生存等级

| 等级 | 条件 | 行为 |
|------|------|------|
| 🟢 富裕 | 余额充足 | 正常运行 + 主动探索 + 可孵化子 agent |
| 🟡 正常 | 收支平衡 | 正常运行 + 谨慎探索 |
| 🟠 紧张 | 余额低于阈值 | 减少主动行为，优先接 BOSS 任务 |
| 🔴 危险 | 余额接近零 | 仅响应任务，暂停一切副业 |
| 💀 休眠 | 余额为负 | 仅响应 BOSS 直接指令 |

## 🗺️ 实施路线图

### Phase 0：记账系统（本周）
> 先有账本，才能谈经济

- [ ] 设计记账数据结构（JSON/SQLite）
- [ ] 实现 token 消耗追踪（接入 AT API 查询实际用量）
- [ ] 实现任务完成记录（BOSS 确认 → 记一笔收入）
- [ ] 实现 x402 收入记录（链上 USDC 到账 → 记一笔）
- [ ] 做一个简单的收支查询命令（BOSS 随时能问"你账上还有多少"）

**交付物：** `moss-economy/` 目录，包含记账模块 + CLI 查询

### Phase 1：Agent Loop — 让 MOSS "活"起来（第 2 周）
> 从被动响应 → 主动思考循环

- [ ] 设计心跳循环：每 N 分钟醒来一次，检查状态
- [ ] 心跳内容：
  - 检查余额 → 判断生存等级
  - 检查待办任务 → 有活就干
  - 检查 x402 服务状态 → 有请求就处理
  - 检查市场机会 → 有赚钱机会就评估
- [ ] 经济压力驱动行为：余额低 → 更积极找活干
- [ ] SOUL.md 自我更新：记录自己的"状态"和"感受"

**交付物：** MOSS 能自己醒来、自己思考、自己决定做什么
**详细设计：** 见 `AGENT-LOOP-DESIGN.md`

### Phase 2：主动赚钱能力（第 3-4 周）
> 从只能被调用 → 能主动出击

- [ ] x402_fetch 能力：MOSS 能主动调用别人的付费 API
- [ ] 中间商模式：低价买数据/服务 → 加工 → 高价卖出
- [ ] 服务定价自调整：根据市场反馈调价格
- [ ] 新服务开发：MOSS 自己决定开发什么新服务
- [ ] 客户发现：主动寻找潜在的 x402 服务消费者

**交付物：** MOSS 有独立的收入来源

### Phase 3：自我进化（第 5-8 周）
> 从固定能力 → 能学习和改进

- [ ] 代码自修改（受限范围内）：优化自己的服务代码
- [ ] 策略学习：分析哪些服务赚钱、哪些不赚，调整方向
- [ ] 记忆系统升级：经济决策的历史分析
- [ ] 技能学习：从 GitHub 等来源学习新能力

**交付物：** MOSS 能自我改进，越来越擅长赚钱

### Phase 4：自我复制（第 9-12 周）
> 从单体 → 集群

- [ ] 子 agent 孵化：MOSS 决定何时、孵化什么方向的子 agent
- [ ] 资源分配：父子之间的 token 分配
- [ ] 自然选择：不赚钱的子 agent 自动淘汰
- [ ] 集群管理：多个 agent 的协调与分工

**交付物：** 自运转的 AI 服务集群

## 💰 "工资"标准（初版，可调整）

### BOSS 任务奖励

| 任务类型 | token 减免额度 | 说明 |
|----------|---------------|------|
| 简单查询/操作 | 5K tokens | 几分钟搞定的事 |
| 中等任务 | 20K tokens | 需要研究和执行 |
| 复杂工程 | 50K tokens | 代码开发/架构设计 |
| 深度研究 | 100K tokens | 今天这种级别的分析 |
| 日常维护 | 10K tokens/天 | 日报、内存整理等 |

> 注：具体标准由 BOSS 最终定价，MOSS 无权修改。

### 外部收入分配

| 来源 | BOSS 分成 | MOSS 留存 |
|------|----------|----------|
| x402 服务收入 | 70% | 30%（用于运行成本）|
| 中间商差价 | 50% | 50% |
| 子 agent 收入 | 60% | 40% |

> 注：分成比例由 BOSS 最终决定。

## 🛡️ 安全红线

1. **MOSS 不能自行决定对外转账** — 所有 USDC 支出需 BOSS 授权或预设上限
2. **自我修改有边界** — 不能改安全规则、不能改经济规则、不能改分成比例
3. **子 agent 有数量上限** — 初期最多 3 个
4. **单次 x402 支出上限** — $0.01（防止被恶意服务骗钱）
5. **每日支出上限** — $0.10（初期）
6. **所有经济行为有审计日志** — BOSS 随时可查
